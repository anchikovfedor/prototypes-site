# Conventions

## Анатомия прототипа

Прототип — отдельное приложение в `apps/<name>/`:

```
apps/<name>/
  package.json        # name: @proto/app-<name>; зависит от @proto/kit
  vite.config.ts      # react() + optimizeDeps.exclude ['@proto/kit']
  tsconfig.json       # extends ../../tsconfig.base.json
  index.html
  src/
    main.tsx          # createRoot + <App/>
    App.tsx           # <PrototypeRoot target=...><Content/></PrototypeRoot>
    screens/          # экраны = VKUI Panel'и
    components/       # локальные компоненты (например, Framer Motion micro-interactions)
    modals/           # ModalPage/ModalCard
  HANDOFF.md          # заметки для разработки
```

Правило изоляции: прототип **не импортирует** код других прототипов. Общее — только из
`@proto/kit`. Так папку `apps/<name>` можно поднять в продуктовый репозиторий.

## Таргеты

- `mobile-app` — весь вьюпорт, платформа iOS/Android, одноколоночные push-переходы,
  скролл внутри AppRoot (`embedded`/`contain`).
- `desktop-web` — VKCOM, полная ширина, многоколоночные раскладки (`SplitLayout`).
- `adaptive` — реагирует на ширину окна.

Таргет задаётся в `App.tsx`: `<PrototypeRoot target="...">`.

**Рамку устройства не рисуем** (решение пользователя, 2026-07-07): никакого мокапа телефона
вокруг прототипа — ни в ките, ни в самих прототипах. Прототип занимает весь вьюпорт;
смотреть в размере устройства = devtools-эмуляция или реальный телефон. Компонент
`PhoneFrame` из `@proto/kit` удалён — не возвращать.

## Минимальные входные данные прототипа

Перед сборкой зафиксировать минимальный набор Figma-артефактов (не угадывать):

- **Per-state макет** — валидный node-id (или ссылка) на **каждое** UI-состояние прототипа
  (напр. свёрнутая сетка + раскрытый экран; пустое/заполненное; до/после). Состояние берём из
  **рендер-макета**, а не реконструируем из initial-кадра анимации — иначе геометрия плывёт.
- **Motion-node** — узел Figma Motion: источник точных easing / длительности / пружин переходов.

Требования к узлам: лежат в **основном файле** (не в branch и не в «🗑️ Draft»); node-id
протухают — если узел не резолвится, переспросить, а не подставлять старый. Значения тянуть
через Figma MCP: `get_design_context` / `get_metadata` / `get_variable_defs`. Ссылки на
состояния и Motion-node фиксируются в `apps/<name>/HANDOFF.md` → секция «Источник / вход».

## Источник значений анимации (строгий гейт)

Все параметры анимации — easing, длительности, задержки/stagger, смещения, размеры, blur,
opacity, поворот, тени, порядок появления — брать **только** из одного из двух источников:

1. **Таймлайн Figma Motion** через `get_motion_context`. Грабли: на корневом Smart-Animate
   символе таймлайн часто пуст (`timelineDurationMs: null`) — вызывать с **`recursive: true`**,
   значения лежат на дочерних узлах в `codeSnippets.motionDev` (`initial` / `animate` /
   `transition`, `times` / `ease`). Loop-демо: активная фаза = `times[1] × durationMs`
   (напр. `0.25 × 2000 = 500 мс`), дальше — hold для зацикливания.
2. **Таблица-спецификация**, если дизайнер её дал.

**Запрещено** подставлять значения «на глаз», по аналогии, из памяти или из прошлых сессий.
Если какого-то значения нет ни в таймлайне, ни в спеке — **остановиться и явно спросить
пользователя**, не подставляя догадку. Любое неизбежное приближение (то, что CSS/движок не
повторяет 1:1) — назвать явно и согласовать.

Длительности и задержки (включая stagger) — по **сетке 50 мс** и диапазонам из
`review-animations/STANDARDS.md` (Short 100–200 / Medium 250–400 / Long 450–600 /
Extra long 700–1000 мс); ревью-гейт флагает отклонения. Некратное значение из
Figma-таймлайна — противоречие источников: остановиться и согласовать с пользователем.

## Воркфлоу реализации

Двухфазный, с жёстким гейтом апрува между статикой и анимацией:

1. **Вход** — зафиксировать Figma-артефакты (см. выше) → проверка: ссылки резолвятся через Figma MCP.
2. **Статика** — собрать вёрстку всех состояний **без анимаций и переходов** → проверка:
   визуальное совпадение с макетами + **апрув пользователя**. Жёсткий гейт: до апрува статики
   анимацию не делаем. Процедура этапа — скилл **`proto-static`** (Figma MCP → статичная
   вёрстка → скриншот-сверка headless Chrome ↔ Figma → запрос апрува).
3. **Анимация** — только после апрува. **Обязательно** свериться со скиллом
   **`emil-design-eng`** (стандарты motion: easing, длительности, физика, перфоманс, a11y)
   **до** реализации. Переходы: VKUI по умолчанию, Framer Motion / WAAPI точечно. Значения —
   строго из таймлайна Figma / спеки (см. «Источник значений анимации»). Проверка: уважает
   `prefers-reduced-motion` и тумблер Motion. Процедура этапа — скилл **`proto-animate`**
   (проверяет гейт апрува статики, ведёт по источникам значений).
4. **Ревью анимаций (жёсткий гейт).** **Обязательно** прогнать скилл **`review-animations`**
   на диффе анимации перед апрувом: `/review-animations` (вендорен в
   `.claude/skills/review-animations/`; у него `disable-model-invocation: true` → вызывается
   **явно** пользователем, либо применяется вручную по `STANDARDS.md`). Проверка: вердикт
   **Approve**, не осталось hard-flags. При **Block** — исправить и перепрогнать; апрув не
   выдаётся, пока вердикт не Approve.
5. **Передача** — скилл **`proto-handoff`**: DoD-чеклист (см. ниже), финализация
   `HANDOFF.md`, `npm run build:site`, деплой с проверкой по контенту (`techContext.md`
   §«Деплой»), опционально обновление публичного зеркала (`scripts/export-public.mjs`).

Этапы 2, 3 и 5 проходят через **свои** этапные скиллы `proto-static` / `proto-animate` /
`proto-handoff` (`.claude/skills/`): в них зафиксированы пошаговые процедуры, а прохождение
фаз отмечается чекбоксами в `apps/<name>/HANDOFF.md` §«Статус фаз».

Motion-скиллы **вендорены в `.claude/skills/`** из **github.com/emilkowalski/skills**:
`emil-design-eng` (стандарты motion, шаг 3), `review-animations` + `STANDARDS.md` (гейт-ревью,
шаг 4). Оба обязательны и не пропускаются. `review-animations` переведён на русский и
адаптирован под проект (проектные триггеры: трассируемость значений, тумблер Motion, сетка
50 мс; таблица длительностей Short/Medium/Long/Extra long) — upstream-синк теперь вручную. Upstream содержит ещё `animation-vocabulary`
(в проект не вендорен). Встроенный `web-animation-design` — вспомогательно. Каталог
**transitions.dev** (`github.com/Jakubantalik/transitions.dev`) — **только референс идей**;
не ставить пишущим CSS-скиллом: его `:root`-токены и `t-*`-классы конфликтуют с VKUI-токенами
и своей дизайн-системой, а переходы экранов уже есть из VKUI.

## Как добавить прототип

Основной путь — скилл **`/new-prototype <имя> <таргет>`**: копирует `apps/_template`,
переименовывает пакет, ставит зависимости и подсказывает регистрацию в лаунчере. Затем:

1. Зарегистрировать в `apps/index/src/prototypes.ts` (если скилл не дописал).
2. Собрать экраны и навигацию (`useStack` для push, `Epic` для табов, `ModalRoot` для шторок).
3. Заполнить `HANDOFF.md`.

Вручную (без скилла): скопировать `apps/_template` → `apps/<name>`, поменять `name` в
`package.json`, `npm install`, дальше шаги 1–3.

## Совместная работа

- Модель: **ветка на прототип → PR в `main`** (прямые пуши в `main` закрыты branch
  protection). CI на PR: typecheck всех workspace + `format:check` + `build:site`.
- Конфликт-поверхность между авторами минимальна (прототипы изолированы в `apps/<name>`):
  общие точки — `apps/index/src/prototypes.ts` (одна строка на прототип) и
  `package-lock.json`. Лечение: `git rebase origin/main`, затем `npm install`
  (пересоберёт лок), закоммитить результат.
- Общий слой (`packages/kit`, `.claude/`, `memory-bank/`, `scripts/`, `.github/`) меняется
  только через ревью владельца — закреплено в `.github/CODEOWNERS`.

## Нейминг

- Имена переменных/функций/компонентов — английские.
- Папка прототипа — kebab-case (`apps/checkout-flow`).
- `nav`-идентификаторы панелей — строковые (`'feed-list'`), совпадают с `activePanel`.

## Шрифты и рендеринг

Применимо ко всем прототипам.

- **Сглаживание (обязательно при кастомных шрифтах).** VKUI **не** задаёт `font-smoothing`,
  поэтому на macOS Chrome текст рендерится субпиксельно — заметно жирнее, чем в Figma
  (там grayscale). Симптом «шрифт по параметрам как в Figma (вес 500), но ощущается толще» =
  именно это. Лечится глобально:
  ```css
  body {
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
  }
  ```
- **Подключение кастомного шрифта (напр. VK Sans Display).** Файлы → `apps/<name>/src/fonts/`,
  `@font-face` в `src/fonts.css` (импортируется из `App.tsx`) с **относительным** `url('./fonts/…')`.
  Важно: относительный путь, **не** абсолютный `/fonts/…` из `public/` — иначе Vite не хеширует
  ассет и не переписывает путь под подпапочный `base`, и на деплое `/<repo>/<name>/` шрифт 404-ит
  (проверено на cards-drum). Маппинг весов: Regular→400, Medium→500 (отдельные файлы, не faux-bold).
- **Файлы шрифтов НЕ в git** (`apps/*/src/fonts/*.ttf` и `apps/*/public/fonts/*.ttf` в
  `.gitignore`): VK Sans Display проприетарный. Локально — положить в `src/fonts/`. **Для
  публичного деплоя** (CI не видит gitignore-файлы) — раскладывать в CI из base64-секретов
  перед сборкой: `.github/workflows/deploy.yml` → шаг «Restore proprietary fonts», секреты
  `VK_SANS_MEDIUM_B64` / `VK_SANS_REGULAR_B64` (`gh secret set … < <(base64 file)`). Без файлов
  прототип молча падает на системный fallback-стек. Файл на gh-pages скачиваем (неизбежно для
  рендера веб-шрифта), но в git/публичное зеркало не попадает.
- **Где применять.** `--vkui--font_family_accent` у VKUI = **системный** стек, не VK Sans. На
  акцентных текстах (заголовки) задавать семейство явно:
  `font-family: 'VK Sans Display', var(--vkui--font_family_base, sans-serif)`.
  Подписи/футноуты в Figma — `fontFamilyBase` (не Display), их семейство обычно не трогаем.

## Definition of done для прототипа

- Вход зафиксирован в `HANDOFF.md`: Figma-состояния + Motion-node (секция «Источник / вход»).
- Статика всех состояний заапрувлена (гейт) до начала анимации.
- Запускается `npm run dev -w apps/<name>` и собирается `npm run build`.
- Переходы между экранами работают и выглядят как задумано.
- Анимации уважают `prefers-reduced-motion` и тумблер Motion.
- Ревью `review-animations` зелёное (вердикт Approve).
- `HANDOFF.md` заполнен: тайминги/easing, что дефолт VKUI vs кастом, маппинг компонентов.
- Строка добавлена в лаунчер (`apps/index`).
